Aprendemos sobre como criar repositórios, e aprendemos que os comandos git clone e git pull são fundamentais para o uso do Git na colaboração em projetos de software.

O comando git clone é utilizado para criar uma cópia local de um repositório remoto. Quando um desenvolvedor deseja iniciar o trabalho em um projeto existente que está hospedado em um repositório remoto, ele pode usar o comando git clone <URL do repositório>. Isso resulta na criação de um novo diretório com o nome do repositório, onde todos os arquivos e o histórico de versões são copiados. Assim, o repositório local fica pronto para ser utilizado, permitindo que o desenvolvedor comece a trabalhar imediatamente.

Por outro lado, o comando git pull serve para atualizar o repositório local com as últimas mudanças do repositório remoto. Ele combina duas operações: git fetch, que baixa as atualizações, e git merge, que integra essas alterações ao branch atual do repositório local. Quando um desenvolvedor deseja obter as últimas alterações feitas por outros colaboradores antes de continuar seu trabalho, ele utiliza o comando git pull origin main, por exemplo. Isso garante que o desenvolvedor esteja sempre trabalhando com a versão mais atual do código. Caso haja conflitos durante a mesclagem, será necessário resolvê-los manualmente.
